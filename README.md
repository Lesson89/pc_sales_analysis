PC Sales Analytics SQL Portfolio Project

This project demonstrates practical SQL skills using a real-world PC sales dataset stored in Microsoft SQL Server.

The objective is to analyze sales performance, investigate data quality issues, and answer business questions through SQL queries commonly used by Data Analysts, Business Intelligence Analysts, and SQL Developers.

Project Goals
Explore and profile sales data
Calculate revenue, cost, profit, and profit margins
Analyze salesperson performance
Rank PC brands by revenue and profit
Investigate data quality issues and anomalies
Perform geographical and payment method analysis
Build reusable SQL components using Views and Stored Procedures
Demonstrate SQL best practices and business-focused analytics
Technologies Used
Microsoft SQL Server
SQL Server Management Studio (SSMS)
T-SQL
SQL Skills Demonstrated
SELECT Statements
WHERE Filtering
Aggregate Functions (SUM, COUNT, AVG, MIN, MAX)
GROUP BY
ORDER BY
CASE Statements
Window Functions (RANK)
Common Table Expressions (CTEs)
Data Quality Analysis
Business Performance Reporting
Views
Stored Procedures
Key Business Questions Answered
How much revenue, cost, and profit were generated?
Which salespeople generated the most revenue?
Which PC brands perform best?
What is the overall profit margin?
Which payment methods have the highest data quality issues?
How many orders have missing ship dates?
Which regions generate the highest revenue?
Key Findings
Total Revenue: 196.8 Million
Total Cost: 161.7 Million
Total Profit: 35.1 Million
Profit Margin: 17.82%
Microsoft was the highest revenue-generating PC brand.
Bank Transfer transactions showed the highest percentage of missing ship dates.
Over 5,000 records contained missing shipping information, highlighting a potential data quality issue.
